# 隐私政策
**应用名称：** Node Expense: Track Together
**平台：** iOS
**最后更新日期：** 2026-03-17

## 1. 概述
Node Expense: Track Together（"我们"、"我们的"）尊重您的隐私。本隐私政策解释了 Node Expense 收集哪些数据、如何使用这些数据，以及您在使用 Node Expense 时有哪些选择。

## 2. 我们收集的数据

### 2.1 您提供的信息
- 支出记录（金额、备注、分类）
- 群组数据（群组名称、成员、设置）
- 群组支出与结算
- 显示名称（用于在共享群组中识别身份）
- 可选的位置数据（仅在您启用时收集）
- 日历访问权限（仅限您选择的日历）

### 2.2 自动收集的数据
- 匿名使用分析（通过 Firebase Analytics）
- 匿名设备标识符——用于在重新安装应用时维护您的订阅和应用数据
- 崩溃日志与诊断信息
- 设备与应用性能信息
- 推送通知令牌——在启用时用于发送群组活动更新和个性化阈值警报

这些数据仅用于提高 Node Expense 的稳定性、性能和用户体验。

### 2.3 订阅与付款
购买和订阅由 Apple 和 RevenueCat 处理。我们不会收到您的信用卡或付款详情。RevenueCat 存储匿名订阅者标识符，以管理您的订阅状态、"终身"访问权限和免费试用期。这些数据将根据需要保留，以便在重新安装设备时验证您的专业版 (PRO) 功能访问权限，并独立于您的群组支出云端数据进行处理。

## 3. 第三方服务
Node Expense 使用以下第三方服务：

- RevenueCat——订阅和应用内购买管理，包括免费试用跟踪
- Google AdMob——横幅广告、激励广告和插屏广告
- Firebase Analytics——匿名使用分析
- Firebase Firestore——群组数据同步（支出、成员、结算、删除申请）
- Firebase Authentication——用于维护订阅连续性的匿名用户识别
- Firebase Cloud Functions——后台任务，包括群组删除和推送通知发送
- Firebase Cloud Messaging（FCM）——群组活动的推送通知

Firebase 仅用作群组相关功能的同步层。个人（非群组）支出数据不会上传到云端服务。

## 4. 广告
Node Expense 为免费层级的用户提供广告支持。这些可能包括横幅广告、达到每月限制时的插屏广告，或用于解锁额外支出名额的可选激励广告。

为了提供更加个性化的体验，我们可能会请求您的跟踪许可。这有助于我们的广告合作伙伴投放更相关的广告并衡量其效果。如果您选择不被跟踪，您仍会看到相同数量的广告，但相关性可能会降低。您的财务数据保持私密，绝不会与广告商共享。

## 5. 位置数据
位置数据是可选的，仅在您在"设置"中明确启用支出追踪时才会收集。启用后，您的位置将随新支出一起记录。位置数据存储在您的设备本地，不会上传到云端服务。您可以随时在 iOS 设置中禁用位置访问权限。

## 6. 日历访问权限
Node Expense 仅访问您选择的日历。日历访问权限仅用于将支出条目显示并同步为日历事件。日历数据绝不会与第三方共享。

## 7. 数据存储、保留与加密

### 7.1 设备端存储
以下数据使用 SwiftData（Apple 的安全本地存储框架）存储在您的设备本地：
- 个人（非群组）支出记录
- 应用设置与偏好
- 用于离线访问的群组数据本地副本
- 用于离线查看的缓存群组支出

除非您明确导出或备份，否则个人（非群组）支出数据绝不会离开您的设备。已删除群组的支出将保留在您的设备本地并带有"已删除群组"标记，以便保存您的记录。

### 7.2 群组功能的云存储
对于群组功能，选定的数据将同步到 Firebase Firestore，以便所有群组成员都能看到相同的信息：

- 群组文档（名称、描述、图标、成员、权限、共享代码）
- 群组支出（金额、分类、备注、日期、位置（若启用）、付款人身份）
- 群组结算（成员间的付款记录）
- 群组删除申请和成员投票
- 试用开始日期（用于重新安装恢复）
- 推送通知令牌（FCM）

云同步仅用于群组功能。个人支出绝不会上传。

### 7.3 加密与备份
- 拥有功能访问权限的用户（专业版订阅或处于有效免费试用期）可以启用使用 AES-256-GCM 加密的加密备份。
- 您可以控制支出记录的保留时长（永久或有限年限）。
- 备份文件会导出到您选择的位置（iCloud 云盘、我的 iPhone 或其他 iOS 文件位置）。
- 备份加密密钥自动管理，且仅存储在您的 iCloud 主钥匙串中，绝不会传输到我们的服务器，开发者也无法访问。该密钥在"密码"应用中不可见，但会通过 iCloud 安全同步到您的其他设备。只有在登录您 Apple ID 的设备上的本应用才能访问密钥以加密或恢复备份。

## 8. 数据迁移
拥有功能访问权限的用户（专业版订阅或处于有效免费试用期）可以以 JSON 格式导出和导入数据。Node Expense 无法访问您导出的文件。导出文件将保存到您选择的位置（iCloud 云盘、我的 iPhone 或其他 iOS 文件位置）。

## 9. 数据删除
您可以通过以下方式删除数据：
- 在 Node Expense 内删除单个或所有个人支出
- 退出或申请删除群组——当所有成员同意时，群组数据将从 Firebase 中移除
- 标记为"删除时保留"的群组支出将保留在您的设备本地，带有已删除群组标记；它们不会被重新上传
- 卸载 Node Expense 将删除设备上存储的所有本地数据

注意：Firebase 群组数据（支出、结算、成员记录）可能会保留，直到通过应用内删除流程正式删除群组。

## 10. 儿童隐私
Node Expense 不适用于 13 岁以下的儿童。我们不会故意收集 13 岁以下儿童的个人数据。如果您认为儿童提供了个人信息，请联系我们。

## 11. 您的权利
您可以：
- 在 iOS 设置中禁用位置访问权限
- 在 iOS 设置中撤销日历访问权限
- 在 Node Expense 内删除您的本地支出数据
- 在删除前导出您的数据
- 申请删除群组（为所有成员从 Firebase 中移除群组数据）
- 随时通过 Apple App Store 设置取消订阅
- 随时停止使用备份或加密导出

## 12. 本政策的变更
我们可能会不时更新本隐私政策。变更将通过本文档顶部的"最后更新日期"反映。变更发布后继续使用 Node Expense 即表示接受更新后的政策。
